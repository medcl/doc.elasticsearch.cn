p. @has_child@ 查询仅仅是将一个 "has_child":has-child-filter.html 过滤器包含进了一个 "constant_score":constant-score-query.html 中. 他的语法跟 "has_child":has-child-filter.html filter 是一样的:

<pre class="prettyprint lang-js">
{
    "has_child" : {
        "type" : "blog_tag"
        "query" : {
            "term" : {
                "tag" : "something"
            }
        }
    }
}    
</pre>

h2. Scope

p. A @_scope@ can be defined on the filter allowing to run facets on the same scope name that will work against the child documents. For example:

<pre class="prettyprint lang-js">
{
    "has_child" : {
        "_scope" : "my_scope",
        "type" : "blog_tag"
        "query" : {
            "term" : {
                "tag" : "something"
            }
        }
    }
}    
</pre>

h1. 内存考量

p. 目前的实现方式是, 所有 @_id@ 的值都被载入了内存(堆)以便于更快的查找, 所以请确认有足够的内存来保存它们.
